# Roadmap

## Near Term

- replay mode hardening for reproducible demos and debugging
- live WebSocket event adapter
- agent timeline with clearer step ordering and transitions
- risk veto visualization for final decision review
- exportable decision trace for audit and sharing

## Medium Term

- optional upstream integration paths that stay low-risk for maintainers
- community plugin packaging for adapters, renderers, and custom workflows
- improved environment setup and CI validation for Docker and manual runs

## Review Goal

The intended trajectory is a project that is easy to evaluate, easy to run, and safe to review independently of any official upstream endorsement.
